Vim, 가장 가볍고, 강력한 에디터

웹표준, 웹접근성을 화두로 몇년전 부터 발전을 거듭해온 웹 퍼블리싱 분야에서 Editor의 선택은 생산성을 좌우하는 중요한 도구로 여겨지고 있으며 단순한 마크업 기능만을 제공해 오던 이전에 비해 Visual C++이나 이클립스 처럼 개발영역에서 제공해 오던 막강한 기능으로 무장하여 그 만큼 선택의 폭을 넓혀 가고 있다.

10년 동안 써 오던 에디트 플러스와의 이별을 준비하며 서브라임 텍스트(Sublime Text)와 압타나 스튜디오(apatana studio) 사이에서 고민을 하던 중 접하게 된 Vim 에디터는 생소함 만큼이나 호기심을 끌기에 충분했다. 윈도우 사용자에겐 거의 알려져 있지 않은 VIM은 사실 오랜 역사만큼이나 전세계에서 가장 많이 알려져 있는 텍스트 에디터이다.

사실 GUI(Graphic User Interface)기반의 편리성과 사용성으로 무장된 요즘에 추세에 비추어 볼때 텍스트 기반의 CUI(Character User Interface) 에디터인 Vim 프로그램은 시대에 역행하는 것 처럼 보이지만 가볍고 확장성이 뛰어나며 마우스 없이 키보드 만으로도 코딩이 가능하다는 사실 만으로도 충분한 매력을 느낄 수 있지 않을까. Vim은 어느 정도의 Learning Curve를 필요로 하지만 한번 빠져들면 결코 헤어나오기 힘든 그 만의 매력을 하나씩 파헤쳐 보고자 한다. 자세히 보기
